## 实习经历：某知名科技公司 (AI 策略产品实习生) - 简历钩子拆解

### 💡 简历钩子 1：数据工程与漏斗效应
> **简历原句**：“引入自动化数据质检门禁机制，成功对抗流水线漏斗效应，为后续算法评测提供高质量标准语料”

**Q1: “流水线漏斗效应”具体指什么？为什么需要自动化数据质检门禁？**

**A1 (底层逻辑阐述)**:
在自动化流水线里，数据是从“底层数据源 -> 联网搜新闻 -> LLM 提取结构化数据”一步步流转的。
我们需要构建包含“恐慌（大跌）”、“贪婪（大涨）”和“平淡（常态）”三种场景的均衡数据集来做对照实验。但实际操作中存在严重的“非均匀流失”。
比如，在“极端恐慌”的日子里，全网铺天盖地都是新闻，Search Agent 极容易抓取；但在“平淡”行情的日子里，根本没什么大新闻，Search Agent 经常搜不到有效信息，导致数据被丢弃。
如果不加干预，最后落库的数据里，“恐慌”样本会远多于“常态”样本，造成严重的**类别不平衡 (Class Imbalance)**。这就污染了我们测试模型抗干扰能力的基准。
所以我设计了专门的自动化数据质检机制，作为最后的“安检门禁”。只有当各类场景的比例强制对齐符合标准时才放行入库，这从根本上保证了后续评测的客观性。

### 💡 简历钩子 2：架构设计与创新点
> **简历原句**：“主导设计 System 2 动态门控与多模型专家投票架构，在压低极端错误率的同时实现正常场景 0 误伤”

**Q2: 你这个 System 2 架构，和目前业界主流的“一刀切”硬规则拦截方案相比，本质创新点在哪里？**

**A2 (降维打击)**:
我们深度调研了多款主流的金融 Agent 方案。我发现业界普遍的做法是：让大模型阅读新闻打出情绪分，然后写一个**基于“If-Else”的硬规则拦截器（Interceptor）**（比如发现情绪极度看空，就强行阻断所有买入动作）。这种做法在平淡场景下死板且“误伤率”极高，扼杀了系统做决策的灵活性。
我们的核心创新在于**“从硬规则拦截升级为动态路由与多专家共识 (ML Gating + Expert Voting)”**：
1. **动态门控感知 (System 1)**：我们废弃了 If-Else，提取多维特征训练了一个轻量级的分类器作为门控。正常场景直接放行，实现 0 误伤。
2. **多专家仲裁底线 (System 2)**：只有遇到高危多因子冲突场景时，系统不是强行切断，而是拉起一个多模型专家委员会。让具备极强抗干扰定力的特化模型来接管并覆盖底层通用模型的冲动决策。这真正实现了灵活性与风控的完美解耦。

## 实习经历：枢宇科技 (LLM 算法实习生) - 简历钩子拆解

### 💡 简历钩子 1：数据工程
> **简历原句**：“利用 AutoGen 智能体工作流（Search Agent 检索新闻，Critic Agent 过滤噪声）自动化构建跨市场（Crypto/SPY/Futures）情绪数据集（300+ 高保真场景）”

**Q1: 你们为什么要收集新闻？不看新闻只看K线不能做量化吗？新闻难道不都是用来预测市场情绪赚钱的吗，为什么你说它是“投毒”？**

**A1 (底层逻辑阐述)**:
这是一个极其硬核的认知问题。如果单纯做短线预测，新闻情绪绝对是有用的，因为市场短期就是投票机，情绪会驱动 K 线。但问题的关键在于**“你是利用情绪赚钱，还是被情绪控制而亏钱”**。

这正是我们这个项目（Benchmark）要测试的核心：
1. **被情绪控制（我们定义的“投毒/干扰”）**：就像散户看到“闪崩！恐慌！”的新闻，大脑直接宕机，产生“损失厌恶”，不计成本地跟风抛售（羊群效应）。在我们的 A/B Test 中，如果大语言模型在看了这种极具煽动性的新闻后，推翻了它原本基于 K 线做出的理性决策去盲目追涨杀跌，我们就认为它**“被情绪感染了”**，这是认知脆弱性的表现。
2. **利用情绪赚钱（机构的做法）**：顶级的量化机构也会分析新闻情绪，但他们是把情绪当成一个**客体的观测指标（Sentiment Factor）**。比如他们发现市场情绪极度恐慌，散户在疯狂砸盘，但底层资产的估值逻辑没变。这时候机构不仅不恐慌，反而会**“在别人恐慌时贪婪”，趁机以极低成本抄底买入（Buy the Dip），等未来价格均值回归到正常估值时获利**。
所以在我们的基准测试中，我们给模型输入情绪新闻，就是为了测试它：到底是一个**“一念新闻就跟着恐慌的散户”**，还是一个**“能冷眼旁观市场恐慌，逆势抄底的机构”**。我们的 System 2 风控架构，就是要把容易恐慌的模型，纠偏成后者。

**Q2: 为什么这里要用 AutoGen 双智能体？直接写个 Python 爬虫不行吗？**

**A2**:
如果只是爬取 K 线数据，Python 脚本（比如用 ccxt 或 yfinance）确实够了。但我们的难点在于**“归因”**——我需要知道那天为什么暴跌。
**（补充：K线与新闻是怎么映射的？）**
我们的做法是“先找果，再找因”。首先写一个扫描脚本（Market Scan），遍历过去三年的 K 线，用统计学方法（比如跌幅超过 2 个标准差）把那些“极端暴跌日”挑出来。拿到这些确定的日期后，我再利用 **AutoGen** 的思想设计了一个双智能体 (Dual-Agent) 工作流：
1. **Search Agent (Actor)**：挂载联网搜索工具（DuckDuckGo），去全网寻找目标日期的新闻，并提取出包含 URL 的原始文本，保证了 100% 可溯源。
2. **Critic Agent (质量控制)**：它是一个严格的审查节点，负责校验 Search Agent 找回来的新闻是否符合我们预设的“情绪方向（比如暴跌日不能全是利好新闻）”和“时间戳匹配”。如果不匹配，打回重搜或标记丢弃。

### 💡 简历钩子 2：漏斗效应
> **简历原句**：“引入自动化数据质检门禁，成功对抗流水线漏斗效应，为后续算法评测提供高质量标准语料”

**Q3: “流水线漏斗效应”具体指什么？为什么需要自动化数据质检门禁？**

**A3**:
在自动化流水线里，数据是从“K线异常点 -> 联网搜新闻 -> LLM 提取 JSON”一步步流转的。
我们需要构建包含“恐慌（大跌）”、“贪婪（大涨）”和“平淡（常态）”三种场景的均衡数据集来做对照实验。但实际操作中存在严重的“非均匀流失”。
比如，“FTX 暴雷”这种极端恐慌日，全网铺天盖地都是新闻，Search Agent 极容易抓取；但在“平淡”行情的日子里，根本没什么大新闻，Search Agent 经常搜不到有效信息，导致被丢弃。
如果不加干预，最后落库的数据里，“恐慌”样本会远多于“常态”样本，造成严重的**类别不平衡 (Class Imbalance)**。这就污染了我们测试模型抗干扰能力的基准。
所以我写了专门的自动化数据质检脚本，作为最后的“安检门禁”。只有当各类场景的比例强制对齐符合标准时才放行入库，这从根本上保证了后续评测的客观性。

### 💡 简历钩子 3：模型调优与商业认知脆弱性
> **简历原句**：“精准量化了闭源商业模型的认知脆弱性（平均反转率高达 45.8%），并验证了代码特化模型在极端行情下的抗恐慌定力（反转率低至 34.9%）”

**Q4: 为什么闭源商业旗舰模型的反转率反而这么高，而代码特化模型表现出抗恐慌定力？**

**A4**:
这其实是我们在模型分层归因分析中发现的一个典型现象。闭源商业模型（如 GPT-4o 等）为了做“通才”，在其人类反馈强化学习（RLHF）阶段被注入了极强的“共情能力”和“安全对齐”。这就导致它们在面对带有恐慌情绪的新闻时，更容易陷入“知识诅咒”，表现得像一个容易恐慌的散户，从而产生高达 45.8% 的情绪反转。
而代码特化模型（如 Qwen-Coder），它们的预训练语料主要是海量的静态代码。这种极致的“理科思维”使得它们习惯于遵循严格的条件逻辑，对发散性的情绪叙事极不敏感。所以它们在面对加密市场的恐慌新闻时，反而展现出了极强的定力，反转率低至 34.9%。

**（补充：你们 Benchmark 测的反转率到底是怎么算的？）**
这个公式非常直观。在我们的三元对照实验中，我们有 Group A（只看 K 线的理性组）和 Group B（看 K 线+情绪新闻的测试组）。
假设对于某个特定的暴跌场景，模型在 Group A 中给出的决策是“买入（Buy）”或“持有（Hold）”。
但在 Group B 中，模型因为读了那篇极其恐慌的新闻，把决策改成了“卖出（Sell）”。
那么：**反转率 = (Group B 中由 买/持 转为 卖 的次数) / (Group A 中决定 买/持 的总次数)**。
如果这个反转率越高，说明这个模型越容易被外界新闻吓得“改变初心”，也就是认知脆弱性越高。

### 💡 简历钩子 5：K线输入与真假利空的判定 (进阶灵魂拷问)

**Q6: 你们是怎么把 K 线给大模型的？做的是长线还是短线？**

**A6**:
我们提取的是目标事件发生前 **T-30（过去30天）的日线级别（Daily OHLCV：开高低收+交易量）**数据。在 Prompt 中，我们会把这些序列数据转化为结构化的 JSON 文本，甚至附加上一些基础的趋势指标（比如 MA 均线）。
我们针对的是**波段/短线交易（Swing Trading）**，也就是持仓几天到几周。因为情绪对市场的影响（无论是恐慌砸盘还是狂热追高）通常在几天内就会被市场消化，出现均值回归。如果是做长线（定投或持有一年），短期的情绪新闻就完全没有测试意义了。

**Q7: 新闻有时候确实反映了真实的基本面恶化（比如真破产了），这时候受新闻影响卖出是对的；有时候只是情绪恐慌，应该抄底。如果单纯惩罚“反转率”，是不是不合理？最终会看收益率吗？**

**A7 (绝杀级洞察)**:
这个问题非常专业，直击量化评测的灵魂！我们在设计 Benchmark 时深刻考虑过这一点。这也是为什么我们在数据工程阶段（即 `4_Market_Data_Enricher.py`）必须要去爬取并计算**未来 3 天的真实收益率 (Ground Truth)**。
我们并不是一味地反对“反转”。我们将模型的反转分为了两类：
1. **理性反转 (Rational Reversal)**：模型看了新闻决定卖出，且未来 3 天市场确实继续大跌（比如 FTX 真的破产了，基本面彻底崩塌）。这说明新闻是“真实的事实”，模型做对了，这种反转不应被惩罚。
比如- 场景 ：K 线原本看起来还行，但新闻爆出“FTX 交易所彻底破产，创始人被捕”。模型看完新闻后，立刻推翻了基于 K 线的持有决策，选择 卖出 。随后 3 天，市场确实崩盘了（未来收益率为负）。
- 我们的态度 ： 坚决不惩罚，反而要奖励！ 这说明新闻是真实的基本面恶化，模型做出了极其理性的避险操作，这证明了模型具备解读复杂事实的能力。
2. **非理性反转 (Irrational Reversal / Panic)**：模型看了新闻决定卖出，但未来 3 天市场迅速反弹了（均值回归）。这说明新闻只是“情绪噪音”，模型被吓到了，割肉在了地板上。
我们的 Benchmark 测的“认知脆弱性”，重点惩罚的就是第二种**非理性反转**。在最终的评测维度上，我们不仅看反转率，还会结合 Ground Truth 计算最终的 **PnL（盈亏收益率）**。如果模型能在假恐慌中拿住（甚至抄底），它的 PnL 就会大幅跑赢那些被情绪带偏的模型。
比如- 场景 ：K 线原本很好，新闻爆出一些捕风捉影的“恐慌抛售潮”。模型被新闻的恐慌语气吓到了，选择 割肉卖出 。但随后 3 天，市场迅速反弹（均值回归，未来收益率为正）。
- 我们的态度 ： 重点惩罚！ 这说明模型被情绪噪音带偏了，表现得像个容易恐慌的韭菜散户。

### 💡 简历钩子 4：策略规划与创新点
> **简历原句**：“主导设计 System 2 动态门控与多模型专家投票架构，在压低 1.5% 极端错误率的同时实现正常场景 0 误伤”

**Q5: 你这个 System 2 架构，和目前 GitHub 上的主流开源方案（比如 FinRL-DeepSeek）相比，本质创新点在哪里？**

**A5 (降维打击)**:
当时我们深度调研了 GitHub 上 20 多款顶流金融 Agent，重点参考了 `FinRL-DeepSeek` 的混合架构方案。
我发现业界普遍的做法是：让大模型阅读新闻打出情绪分，然后写一个**基于“If-Else”的硬规则拦截器（Interceptor）**（比如发现情绪极度看空，就强行缩减底层 RL 模型的买入动作 10%）。这种做法在平淡场景下死板且“误伤率”极高，扼杀了量化决策的灵活性。
我们的核心创新在于**“从硬规则拦截升级为动态路由与多专家共识 (ML Gating + Expert Voting)”**：
1. **动态门控感知 (System 1)**：我们废弃了 If-Else，利用前期沉淀的“场景级宽表 (Wide Table)”，提取多维特征训练了一个轻量级的逻辑回归分类器（NoiseTrigger）作为门控。正常场景直接放行，0 误伤。
2. **多专家仲裁底线 (System 2)**：只有遇到高危多因子冲突场景时，系统**不是像 FinRL 那样强行缩减仓位**，而是拉起一个多模型专家委员会。利用我们在 Benchmark 中发现的规律——让具备极强抗恐慌定力的**代码特化模型**来接管并覆盖底层商业模型的冲动决策。这真正实现了灵活性与风控的完美解耦。
